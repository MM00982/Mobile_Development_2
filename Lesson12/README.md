# Практическая работа №12

## Глава 1. Основные способы отображения списков

В рамках данной работы был создан учебный проект ListsApp для демонстрации трех основных способов реализации прокручиваемых списков в Android.
1.1. ScrollView (Линейная прокрутка)
ScrollView — это контейнер, который позволяет прокручивать свое содержимое. Он подходит для экранов 
с фиксированным набором элементов, которые не помещаются на дисплее. В примере была программно сгенерирована 
геометрическая прогрессия (100 элементов) и добавлена в LinearLayout внутри ScrollView. Особенность этого 
подхода в том, что все элементы создаются сразу и хранятся в памяти, что неэффективно для больших данных.

1.2. ListView (Устаревший способ)
ListView использовался в ранних версиях Android. Он поддерживает прокрутку и использование адаптеров, но имеет 
ограниченные возможности по кастомизации и анимации. В задании реализован простой список книг (строк), заполненный 
с помощью стандартного ArrayAdapter и макета simple_list_item_1.

<img width="2560" height="1535" alt="image_2025-12-20_00-24-51" src="https://github.com/user-attachments/assets/50cfb5ad-263a-4edc-9885-83d7d8ba2359" />

1.3. RecyclerView (Современный способ)
RecyclerView — наиболее мощный инструмент для списков. Он использует паттерн ViewHolder для переиспользования ячеек 
списка (View), что позволяет отображать тысячи элементов без тормозов. В работе создан список "Исторических событий" 
с использованием кастомного адаптера HistoryAdapter и карточек CardView с иконкой и текстом.

## Глава 2. Контрольное задание (Travel Weather)

В основном приложении «Travel Weather» реализованы все требования контрольного задания, включая перевод списка городов на RecyclerView.
Реализация RecyclerView в Travel Weather:
Старый подход с ScrollView был заменен на RecyclerView с LinearLayoutManager.
Создан класс CityAdapter (наследуется от RecyclerView.Adapter), который управляет отображением списка избранных городов.
В MainActivity список городов получается из ViewModel (LiveData) и передается в адаптер.
Реализовано удаление элементов из списка и клик для перехода к прогнозу.

Выполнение функциональных требований:
1. Авторизация: Вход через Firebase Auth (email/пароль), данные пользователя отображаются в профиле.

## Вывод

В ходе работы освоены принципы создания списков в Android. Сравнительный анализ показал, что RecyclerView 
является стандартом для динамических данных благодаря своей производительности и гибкости. В проекте Travel 
Weather список городов был успешно мигрирован на RecyclerView, а архитектура приложения доведена до соответствия требованиям MVVM + Clean Architecture.
